= Practical Software Architecture Patterns with Java
:toc: auto

This repository contains practical Java examples that illustrate software architecture concepts, patterns, and modern AI-driven designs, focusing on real-world trade-offs and system evolution.

== Getting Started

Clone the repository and build the project:

[source,bash]
----
git clone git@github.com:bpbpublications/Practical-Software-Architecture-Patterns-with-Java.git
cd Practical-Software-Architecture-Patterns-with-Java/
mvn clean install
----

Run specific examples by navigating to each module.

== Project Structure

*Part 1 – Foundations*: Core principles of software architecture, covering abstraction, system structure, boundaries, and domain modeling to manage complexity and align with business needs.
*Part 2 – Architecture Patterns*: Practical patterns such as MVC, Hexagonal Architecture, monoliths, microservices, REST, and event-driven systems, including real-world trade-offs and Java implementations.
*Part 3 – AI for Architects*: Modern AI-driven architecture concepts, including LLMs, embeddings, RAG, MCP, and agentic systems, focusing on how AI reshapes system design.


== What You’ll Find Here

- *Chapter 1 – Software Engineering, Architecture, and Abstraction*: Understand architecture as decision-making, focusing on structure, trade-offs, and system evolution.
- *Chapter 2 – Software Design Principles*: Learn how to structure code with clear boundaries, reduce coupling, and apply SOLID principles.
- *Chapter 3 – Documentation and Domain Modeling*: Explore architecture communication and Domain-Driven Design to align software with business needs.

- *Chapter 4 – MVC*: Apply the Model–View–Controller pattern to structure user-facing applications.
- *Chapter 5 – Hexagonal Architecture*: Isolate business logic using ports and adapters for flexibility and testability.
- *Chapter 6 – API Communication Patterns*: Design APIs using REST, gRPC, and WebSockets with clear contracts and trade-offs.
- *Chapter 7 – Event-Driven Architecture*: Build scalable systems with asynchronous communication and event-based design.
- *Chapter 8 – Monoliths and Microservices*: Compare architectural styles and make informed decisions based on context.

- *Chapter 9 – System Execution Models*: Understand batch processing, workflows, and long-running processes in production systems.
- *Chapter 10 – Testing Strategies*: Validate systems using unit, BDD, and integration testing while enforcing architecture.

- *Chapter 11 – AI for Architects*: Learn core AI concepts and how probabilistic systems impact architecture decisions.
- *Chapter 12 – Agentic Architectures*: Design systems with AI agents using reasoning and action loops.
- *Chapter 13 – RAG*: Integrate external knowledge into AI systems using retrieval-based patterns.
- *Chapter 14 – MCP*: Standardize AI integration with external tools and services.


== Requirements

To run the examples in this repository, you will need:

- *Java 21*: Required to compile and run all code samples.
- *Maven*: Used for dependency management and project builds.
- *Docker*: Optional but recommended for running supporting services and environments.
- *Git*: Required to clone and manage the repository.
