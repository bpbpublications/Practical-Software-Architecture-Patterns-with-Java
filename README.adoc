= Practical Software Architecture Patterns with Java
:toc: auto

This repository contains practical Java examples that illustrate software architecture concepts, patterns, and modern AI-driven designs, focusing on real-world trade-offs and system evolution.

== Getting Started

Clone the repository and build the project:

[source,bash]
----
git clone git@github.com:bpbpublications/Practical-Software-Architecture-Patterns-with-Java.git
cd Practical-Software-Architecture-Patterns-with-Java/
mvn clean install
----

Run specific examples by navigating to each module.

== Project Structure

*Part 1 – Foundations*: Core principles of software architecture, covering abstraction, system structure, boundaries, and domain modeling to manage complexity and align with business needs.

*Part 2 – Architecture Patterns*: Practical patterns such as MVC, Hexagonal Architecture, monoliths, microservices, REST, and event-driven systems, including real-world trade-offs and Java implementations.

*Part 3 – AI for Architects*: Modern AI-driven architecture concepts, including LLMs, embeddings, RAG, MCP, and agentic systems, focusing on how AI reshapes system design.

== Requirements

To run the examples in this repository, you will need:

- *Java 21*: Required to compile and run all code samples.
- *Maven*: Used for dependency management and project builds.
- *Docker*: Optional but recommended for running supporting services and environments.
- *Git*: Required to clone and manage the repository.
